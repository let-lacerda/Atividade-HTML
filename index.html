Perguntas e respostas:

  1. Na tag "input" com atributo type="text", para que serve o atributo "placeholder"?

    2. O atributo "placeholder" em uma tag "input" com o atributo "type=text" é usado para fornecer uma dica ou um exemplo do tipo de informação que deve ser inserida no campo de texto. O texto definido no atributo "placeholder" aparece em cinza claro dentro do campo de texto, e desaparece quando o usuário começa a digitar no campo.

  2. Na tag "input" com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?

    3. O atributo "checked" em uma tag "input" com o atributo "type=radio" ou "type=checkbox" é usado para definir o estado inicial do botão de opção ou caixa de seleção como selecionado. Quando o valor do atributo "checked" é definido como "checked", o botão de opção ou caixa de seleção é exibido inicialmente como selecionado. Se o usuário desejar desmarcar o botão de opção ou caixa de seleção, ele poderá clicar nele para alternar o estado.

  3. Na tag "input" para que serve o atributo "value"?

    4. O atributo "value" em uma tag "input" é usado para definir o valor associado ao elemento de formulário, que é enviado ao servidor quando o formulário é enviado. O valor do atributo "value" depende do tipo de entrada de formulário especificado no atributo "type".

  4. Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

    5. Para agrupar vários botões de rádio em um formulário de modo que apenas um deles possa ser selecionado de cada vez, é necessário atribuir o mesmo valor ao atributo "name" de cada botão de rádio. Isso garante que todos os botões de rádio pertencem ao mesmo grupo e apenas um pode ser selecionado de cada vez.

  5. Para que serve a tag "label"? Para que serve o atributo "for" nessa tag?

    6. A tag "label" em HTML é usada para associar um rótulo a um elemento de formulário, como um campo de texto, botão ou caixa de seleção. O rótulo é exibido como texto próximo ao elemento de formulário e, quando o usuário clica no rótulo, o elemento de formulário correspondente recebe o foco ou é selecionado. O atributo "for" na tag "label" é usado para associar o rótulo a um elemento de formulário específico. O valor do atributo "for" deve corresponder ao valor do atributo "id" do elemento de formulário correspondente.

  6. Na tag "option" para que serve o atributo "selected"?

    7. O atributo "selected" é usado na tag "option" em um elemento "select" para definir a opção selecionada por padrão quando a página é carregada. A opção com o atributo "selected" definido é exibida como a opção padrão no menu suspenso.

  7. Nas tags de formulário para que serve o atributo "required"?

    8. O atributo "required" é usado em elementos de formulário em HTML (como "input", "select" e "textarea") para indicar que um determinado campo deve ser preenchido pelo usuário antes que o formulário possa ser enviado. Quando um campo de formulário é marcado com o atributo "required", o navegador não permitirá que o formulário seja enviado até que o campo seja preenchido. Se o usuário tentar enviar o formulário sem preencher o campo obrigatório, o navegador exibirá uma mensagem de erro solicitando que o usuário preencha o campo.

  8. Para que serve a tag "textarea"?

    9. A tag "textarea" em HTML é usada para criar uma caixa de texto de várias linhas, permitindo que o usuário insira texto em um formulário ou página da web. A caixa de texto pode ser dimensionada horizontalmente e verticalmente usando os atributos "cols" e "rows" da tag "textarea".

  9. Na tag "form" para que serve o atributo "action"?

    10. O atributo "action" na tag "form" em HTML é usado para especificar a URL do script ou programa que irá processar os dados do formulário quando o usuário o enviar. Em outras palavras, ele define o destino para onde os dados do formulário serão enviados quando o usuário clicar no botão de envio do formulário. O valor do atributo "action" é geralmente uma URL relativa ou absoluta que aponta para um arquivo do lado do servidor que pode processar os dados enviados pelo formulário. Esse arquivo do lado do servidor pode ser escrito em uma linguagem de programação como PHP, Python, Ruby, etc.

  10. Na tag "form" para que serve o atributo "method"?

    11. O atributo "method" na tag "form" em HTML é usado para especificar o método HTTP que deve ser usado para enviar os dados do formulário para o servidor. Existem dois métodos HTTP que podem ser usados para enviar dados de formulário: "GET" e "POST". O método "GET" envia os dados do formulário como parte da URL da solicitação. Isso significa que os dados do formulário são visíveis na barra de endereço do navegador. O método "GET" é geralmente usado para enviar dados simples do formulário, como pesquisas ou parâmetros de filtro. O método "POST" envia os dados do formulário no corpo da solicitação HTTP, o que significa que os dados do formulário não são visíveis na barra de endereço do navegador. O método "POST" é geralmente usado para enviar dados mais complexos do formulário, como formulários de inscrição, envio de arquivos, etc.

  11. Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag "form"?

    12. O atributo "method" na tag "form" em HTML é usado para especificar o método HTTP que deve ser usado para enviar os dados do formulário para o servidor. No entanto, o valor do atributo "method" não tem impacto na exibição do valor digitado de uma senha na barra de endereços do navegador. Para ocultar o valor digitado de uma senha na barra de endereços do navegador, é necessário definir o atributo "type" do elemento "input" como "password". Isso fará com que o campo de entrada de senha exiba apenas asteriscos ou pontos em vez do valor real da senha.

  12. Qual valor deve ser informado no atributo "method" na tag "form" em um formulário que será usado para fazer upload de arquivos?

    13. O valor do atributo "method" na tag "form" em um formulário que será usado para fazer upload de arquivos deve ser definido como "POST". Isso ocorre porque o envio de arquivos através de formulários HTML requer o uso do método "POST" para enviar o arquivo para o servidor. O método "GET" não pode ser usado para envio de arquivos, pois ele limita o tamanho da solicitação, e os arquivos geralmente são muito grandes para serem enviados dessa maneira.

  13. O que as tags "fieldset" e "legend> produzem?

    14. As tags HTML "fieldset" e "legend" são usadas em conjunto para agrupar elementos de formulário relacionados e fornecer uma legenda para esse grupo. A tag "fieldset" é usada para criar uma caixa de grupo que envolve vários elementos de formulário relacionados. Isso ajuda a organizar e agrupar os campos de formulário que têm uma finalidade semelhante. O "fieldset" pode conter qualquer elemento de formulário, como "input", "label", "select", "textarea", entre outros. A tag "legend" é usada para fornecer uma legenda para o grupo de campos de formulário contidos no elemento "fieldset". A legenda geralmente descreve o propósito dos campos de formulário contidos no "fieldset". A tag "legend" deve ser colocada logo após a tag de abertura "fieldset".
